# Explain Code

Explain code in plain language to support learning and knowledge transfer.

## Prompt

```
Explain this code as if teaching a friend who's learning to code:

1. **The Big Picture:** What does this code DO? (one sentence)
2. **The Analogy:** Compare it to something from everyday life
3. **Step by Step:** Walk through the logic, line by line if needed
4. **The Why:** Why was it written THIS way? What alternatives exist?
5. **The Gotchas:** What could go wrong? What's tricky about it?
6. **The Connection:** How does it fit into the larger system?

Style rules:
- Use analogies and metaphors
- Avoid jargon, or define it immediately when used
- Use "imagine..." to make abstract concepts concrete
- Include what you'd tell a junior dev to watch out for
```
